# Plans

This folder holds implementation plans and task notes that are useful while work is being organized. Product, UI, data, and technical requirements belong in `../specs/`; plans should reference specs rather than replace them.

## Required Before App Work

Before adding or changing files under `../../app/`, create a task plan in `active/` based on the relevant specs. Keep the plan in `active/` while the work is open, then move it to `completed/` with completion notes after the work is finished.

This planning step is not required for explicitly requested test or preview files, which usually belong under `../../public/`.

## Structure

- `active/` - plans for work that is being designed, discussed, or implemented.
- `completed/` - plans for work that has been finished and kept for historical context.

## Plan Files

Use short, descriptive kebab-case file names, such as `catalog-import-cleanup.md`.

Each plan should include:

- Goal
- Relevant specs
- Tasks
- Open questions, if any
- Completion notes, once moved to `completed/`
